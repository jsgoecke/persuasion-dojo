## Vision

### 10x Check
A platform for developing human influence skills in an AI world. Every meeting you've
ever had is grist for the system — retroactive analysis builds your Persuasion Score
history before you even installed the app. The relationship graph covers everyone you
interact with professionally, and profiles compound across your whole team. Before
every high-stakes meeting, the system briefs you on who's in the room and what has
worked with them before. Skill badges mark when you've genuinely internalized a
technique — the system knows because the corresponding coaching prompt has stopped
firing. The product gets better the more you use it, and the data moat compounds.

### Platonic Ideal
The coaching becomes invisible. After 100 meetings, you've internalized most of it.
The prompts feel like intuitions, not notifications. The retrospective feels like
watching tape with a coach who knows your game deeply. The system knows that your
best meetings start with 5 minutes of listening before your first assertion — and it
notices when you skip that. The relationship graph means you arrive at every meeting
knowing exactly who you're talking to and what will land. The AI sparring partner
lets you practice before the real thing. The team intelligence layer makes your entire
sales team measurably more effective as a unit.

## Accepted Scope (full MVP)

### Core Platform (from design doc)
- ScreenCaptureKit audio capture — any meeting platform (Zoom, Teams, Meet, Webex, browser)
- Deepgram streaming transcription with speaker diarization (~300ms latency)
- Participant Superpower profiler (Architect / Firestarter / Inquisitor / Bridge Builder)
- ELM audience state detection (ego-threatened / shortcut mode / consensus-protecting)
- Coaching engine: Claude Haiku real-time, three-layer prompts (self / audience / group)
- Electron overlay (always-on-top, draggable, dismissable, history tray, layer badge)
  - **Position:** User-positionable (draggable). Default: bottom-right corner of screen.
  - **Size:** 260px wide × 90px collapsed, expands to ~210px tall when history tray opens.
  - **Layer hierarchy:** One dominant prompt shown at full size (priority: Audience > Self > Group). Other two layers collapse to single-word labels with chevrons (e.g., "Self →" "Group →"). Tap to expand any layer.
  - **Layer badge:** Small pill in top-left corner of overlay indicating active layer ("Audience", "Self", "Group").
- Persuasion Score (Timing 30% / Ego Safety 30% / Convergence 40%)
- Session debrief / retrospective layer
- Relationship intelligence graph (Superpower + ELM patterns + historical effectiveness)
- Pre-seeding (free-text description or paste)
- Growth Score (longitudinal prompt-frequency-decay + convergence trend)
- Packaged macOS app (Electron + PyInstaller, signed + notarized .dmg)
- Individual + team billing (Stripe)

### Accepted Expansions
1. **Pre-meeting briefing** — Calendar integration detects upcoming meetings; 30-min prior macOS system notification fires ("Board meeting in 30 min — 3 participants profiled"). Click opens a compact briefing window: one card per participant with Superpower type, typical ELM pattern, and suggested opening frame. Window auto-closes when the meeting starts. Requires calendar OAuth (Google Calendar / Outlook).
2. **Persuasion Replay** — Session debrief highlights 3-5 pivot moments on a horizontal visual timeline: where conversation shifted, which prompt fired (or should have fired), what happened to participation distribution afterward. Click any marker to jump to that utterance in the transcript view below (transcript scrolls and highlights the relevant line). No audio playback in V1 — text/visual only.
3. **Skill Badges** — When a coaching prompt type has not fired for 3 consecutive meetings, the system awards a skill badge ("Strategic Silence: the skill is yours now"). Derived directly from prompt frequency decay data already tracked for Growth Score.
4. **AI Sparring Partner** — Before a high-stakes meeting, the user opens a practice session via setup screen: 4 Superpower type buttons (Architect / Firestarter / Inquisitor / Bridge Builder) each with a 1-line description, plus an optional "Meeting context" text field (e.g., "budget review with skeptical CFO"). User selects a type and hits "Start Sparring." Same coaching engine as live meetings. AI opponent maintains character consistency for the session duration. User gets real-time prompts. AI opponent responses stream as they're generated (<3s target).
5. **Retroactive Analysis** — Import Zoom cloud recordings, Otter transcripts, or uploaded audio/video files. System processes them through the full pipeline (transcription → profiler → ELM detection → convergence analysis) to build Persuasion Score history and participant profiles retroactively.
6. **Team Intelligence (V1: read-only shared profiles)** — Team members share participant profiles via JSON export/import (SQLite file sync via iCloud Drive or shared folder). V1 is read-only: profiles are reference data, not editable by team members. Overlay shows a "Team says: Inquisitor (3 observations)" tag for shared profiles alongside the user's own live classification. Team analytics dashboard (aggregate technique effectiveness, peer benchmarking) is V2, gated on individual validation. **UX guardrail:** shared profiles display with a "reference" badge — the coaching engine uses the user's own live observations as the primary input, not team-imported data, to avoid anchoring bias. Enables the enterprise pricing conversation without requiring cloud infrastructure in V1.

## Overlay UX Specification

```
┌─────────────────────────────────────────┐
│ [Audience ▲]              [✕] [⊟ hide] │  ← layer badge + controls
├─────────────────────────────────────────┤
│ Let her feel the problem.               │  ← dominant prompt (primary)
│ Don't drop the data yet.                │
├─────────────────────────────────────────┤
│ Self →          Group →                 │  ← collapsed secondary layers
└─────────────────────────────────────────┘
  260px wide × 90px (collapsed)

  STATES:
  ┌──────────────────────┐  ┌──────────────────────┐
  │ [Audience ▲]    ↻ ✕ │  │ [Audience ▲]    ↻ ✕ │
  │ Learning Sarah...   │  │ ↻ cached        ↻ ✕ │
  │ (building profile)  │  │ Let her feel the     │
  └──────────────────────┘  │ problem.             │
  "Learning" state          └──────────────────────┘
                            Cached fallback state (↻ badge)

  ┌──────────────────────┐  ┌──────────────────────┐
  │ [Audience ▲]    ↻ ✕ │  │ [⚡ Reconnecting...] │
  │ Let her feel the    │  │ Prompts queued (2)   │
  │ problem.            │  └──────────────────────┘
  ├─────────────────────┤  Deepgram disconnect state
  │ ▼ History (3)       │
  │ · anchor in one #   │
  │ · ask a question    │
  └──────────────────────┘
  History tray open
```

**Overlay design decisions:**
- **Position:** Draggable by user. Default: bottom-right, 20px from screen edge.
- **Collapse:** Click [⊟] to minimize to a 40px pill (layer badge only). Click to re-expand.
- **Dismiss:** Click [✕] to dismiss current prompt. Overlay remains visible for next prompt.
- **Layer priority:** Audience > Self > Group. Dominant layer shown at full size; others as single-word labels with →.
- **Reconnect state:** Prompt area replaced by "⚡ Reconnecting..." with queued prompt count badge.
- **Learning state:** Shown for new participants with <7 utterances (confidence <0.70). No prompt fires; overlay shows "Learning [Name]..." with a subtle pulse animation.
- **Cached badge:** `↻ cached` shown in subdued color when Haiku timeout fires fallback. Does not replace prompt text — appears as a badge above or inline with it.

## Accessibility & Keyboard Control

**Global hotkeys (registered system-wide via Electron globalShortcut — works even when Zoom has focus):**
- `⌘+Shift+D` — dismiss current prompt
- `⌘+Shift+L` — cycle through layers (Audience → Self → Group → back)
- `⌘+Shift+H` — toggle history tray open/closed
- `⌘+Shift+M` — minimize/restore overlay (between full and pill mode)
- All hotkeys user-configurable in settings.

**macOS accessibility:**
- Overlay has ARIA role="status" (live region) — screen reader announces new prompts automatically.
- Minimum touch/click target: 44pt for all interactive controls on the overlay.
- Color contrast: prompt text (#ffffff on #1a1a1a) → 18.1:1 contrast ratio (AAA).
- Cached badge (#888888 on #1a1a1a) → 4.6:1 (AA). Ensure badge is also distinguished by shape/icon, not color alone.
- All settings screens: full keyboard navigation (tab order, focus indicators).
- First-run wizard: each step focusable and operable via keyboard alone.

**Multi-monitor:** Overlay remembers its position per-monitor (stored in SQLite user settings). If a monitor is disconnected, overlay defaults to primary display bottom-right.

## Design Anti-Patterns to Avoid

**UI classifier: APP UI** (utility-first, not marketing). Apply App UI rules throughout.

**Debrief view:** Typography-first, not card-first.
- ✅ Persuasion Score as a large number (e.g., 74) with subscores below it in smaller text
- ✅ Persuasion Replay as a horizontal timeline with labeled pivot markers
- ✅ Utility copy: "3 prompts fired" / "Sarah: Inquisitor (0.84)" / "18 min active"
- ❌ NOT: metric-per-card grid (Persuasion Score card + Growth Score card + Prompt Count card)
- ❌ NOT: icons in colored circles as section decoration
- ❌ NOT: centered text alignment throughout
- ❌ NOT: "Your coaching journey today" — name the data, not the feeling

**Overlay:** Minimal chrome, maximum signal.
- ✅ Dark semi-transparent background (not pure black, not white card)
- ✅ Monospaced or high-legibility sans-serif for prompts (readable at a glance)
- ✅ Layer badge as a small pill, not a header section
- ❌ NOT: rounded card with drop shadow (reads as notification, not overlay)
- ❌ NOT: full-opacity white background (too visually heavy on top of meeting window)

**Copy rules:**
- Prompts are forward-looking actions: "Let her feel the problem." / "Anchor in one number."
- NOT observations: "You've been in advocate mode." / "Sarah seems challenged."
- Debrief is data, not praise: "74" not "Great session!"

## Architecture Notes (for Sections 1-11)

The 6 accepted expansions change the architecture in three important ways:

**1. Pre-meeting briefing requires a persistent background service**
The macOS app needs to run in the background (menu bar icon) to fire 30-min pre-meeting
notifications. This is an architectural shift from "open for meetings, close after" to
"always-on background agent."

**2. AI Sparring Partner is a new conversation mode**
The coaching engine needs a "simulation mode" where instead of processing real audio,
it processes a conversational exchange with an AI opponent. The AI opponent maintains
a Superpower type consistently. This is a new execution path alongside the real-time
path.

**3. Team Intelligence requires a sync layer — V1 uses SQLite file sync**
Individual SQLite is fine for solo use. V1 team intelligence uses SQLite file sync
(iCloud Drive / shared folder) rather than a custom backend — profiles are exported
as JSON, team members import them. This is lower fidelity but shippable. A proper
cloud sync backend is V2, gated on: (a) team feature validation, (b) SOC 2 prep.
Team profiles are append-only in V1: all observations from all team members are
preserved, and the overlay shows a confidence range (not a single label) when
team members' observations diverge. True conflict resolution UI is V2.

**4. ScreenCaptureKit PoC — HARD GATE before audio architecture commits**
ScreenCaptureKit captures system audio, but Zoom may route incoming participant
audio separately from system audio. This is a critical unvalidated assumption.

**Build order (CRITICAL — outside voice identified inversion):**
macOS 14+ requires Screen Recording permission, which requires a notarized app.
The PoC cannot be tested without a notarized (or at minimum dev-signed) build.
**Sequence: notarization CI pipeline → PoC. These are not parallel tracks.**
Use development signing for the PoC on your own machine; full notarization is
required before distribution.

**Acceptance criteria for PoC (2 hrs):**
- ScreenCaptureKit captures BOTH outgoing (user mic) and incoming (other participants)
  audio in a live Zoom call, Teams call, and browser-based call
- Combined audio stream can be piped to Deepgram with speaker diarization enabled
- Round-trip latency from speech to Deepgram `is_final` remains <500ms
- **Diarization accuracy ≥85%** correct speaker attribution in a 3-person test call.
  Diarization on a single mixed system audio stream degrades vs. separate mic inputs.
  If <85%: evaluate BlackHole stereo split (user mic left / system audio right) as
  fallback before committing to ScreenCaptureKit.

**If PoC fails:** Revert to BlackHole + AudioSource interface (BlackHole as V1
implementation, ScreenCaptureKit as V2 once the capture issue is resolved or worked
around). Do NOT proceed to Retroactive Analysis, Team Intelligence, or multi-platform
claims until PoC passes.

**Named pipe buffer design (Swift → Python):**
The Swift binary → named pipe → Python path is blocking IPC with no buffering.
At 300ms latency targets, Python slowness or crash drops audio frames silently.
Plan must include: circular audio buffer in Swift binary (500ms), reconnect
protocol if Python side drops, dropped-frame counter for observability.

**Notarization pipeline:** Add to pre-shipping checklist: Electron + PyInstaller
notarization pipeline validated on CI (GitHub Actions); a signed and notarized
dummy .dmg must succeed before any release build is shipped. Hardened runtime
entitlements for microphone + screen recording must be explicitly declared in
the app's entitlements.plist.

**5. GTM constraint — Corporate MDM**
V1 targets users on personal Macs or with IT admin access to grant Screen Recording
permission. Corporate MDM policies on managed MacBooks frequently block permission
grants for non-whitelisted apps. MDM configuration profile (for enterprise IT deploy)
is V2, gated on: (a) first enterprise customer, (b) SOC 2 prep.
**V1 GTM framing:** "Works on your personal Mac. Enterprise MDM support coming."

**6. Convergence spike failure path**
If the convergence validation spike (5-10 annotated transcripts) does not reach
≥75% signal agreement: Persuasion Score is replaced with a simpler Session Summary
(what prompts fired, when, participation distribution). The Growth Score and debrief
remain — only the convergence-dependent scoring components are cut. This is the
contingency plan; the spike is the gate.

**7. Team Intelligence participant identity**
V1 JSON export/import must use email as the primary participant identifier (not name).
Name-matching is ambiguous and corrupts the graph when two team members profile
different "Sarah Chen" participants. If email is unavailable (pre-seed only, no calendar),
fall back to name + company tuple. Add identity resolution design to profiler spec.

**8. API cost model**
Before shipping Stripe billing, model the per-user Haiku API cost:
- Assume 3 hours of active meeting time per day per user
- Prompt fires every 90 seconds on average → ~120 calls/day/user
- Haiku: $0.80/1M input tokens, $4/1M output; ~300 tokens/call
- Cost: ~$0.015/user/day → ~$0.45/user/month
- At $200/month target price, Haiku cost is ~0.2% of revenue. Acceptable.
- Claude Opus for debrief: ~4,000 tokens/session × 3 sessions/day = ~12K tokens/day
  → ~$0.18/user/day → ~$5.40/user/month. At $200/month: ~2.7%. Acceptable.
- **Total API cost at MVP pricing: ~$6/user/month. Healthy margin.**

**9. Retroactive Analysis — V1 scope: uploaded audio files only**
V1 retroactive analysis accepts user-uploaded audio/video files (.mp3, .mp4, .m4a,
.wav) processed through the same Deepgram pipeline as live sessions. Zoom cloud
recording import (requires OAuth) and Otter transcript import (third-party dependency,
copyright risk) are both deferred to V2.
